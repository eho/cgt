# Documentation

This directory is the source of truth for project documentation.

- [Vision](vision/) - strategic vision, product goals, and project direction
- [Architecture](architecture/) - current system architecture, data contracts, and technical decisions
- [Current Design](architecture/current-design/) - how the implemented system works now
- [Design](design/) - active feature specs and implementation plans
- [Planning](planning/) - roadmap, assessments, milestones, migrations, and ongoing planning notes
- [Testing](testing/) - QA guides, testing strategy, manual validation, and verification checklists
- [Operational](operational/) - runbooks, local setup, deployment, CLI usage, and troubleshooting
